# 技术债评估技能

分析代码库中的技术债务，识别问题并生成评估报告。

## 工作流程

### Step 1: 理解评估范围

与用户确认：
1. 评估的目录范围（整个项目 or 特定模块）
2. 关注的问题类型（代码质量、架构问题、性能、安全等）
3. 输出格式（报告、评分、改进建议）

### Step 2: 扫描代码库

使用以下工具分析代码：

```bash
# 代码复杂度分析
npx eslint . --format json --output-file eslint-report.json

# 类型检查（TypeScript）
npx tsc --noEmit

# 依赖分析
npm audit

# 重复代码检测
npx jscpd . --reporters json
```

### Step 3: 分析问题

分类整理发现的问题：
- **严重 (Critical)**: 安全漏洞、数据丢失风险
- **重要 (Major)**: 架构问题、性能瓶颈
- **一般 (Minor)**: 代码风格、小优化

### Step 4: 生成评估报告

创建包含以下内容的报告：

1. **概览**: 项目基本信息、评估时间、总体评分
2. **问题清单**: 按严重程度分类的问题列表
3. **技术债量化**: 修复所需工作量估算
4. **改进建议**: 优先级排序的重构建议
5. **时间规划**: 建议的修复时间表

### Step 5: 输出结果

将报告保存到文件，并提供改进建议。

## 评估维度

| 维度 | 检查项 |
|------|--------|
| **代码质量** | ESLint 错误、复杂度、重复代码 |
| **类型安全** | TypeScript 错误、any 类型使用 |
| **依赖管理** | 漏洞、过期依赖、未使用依赖 |
| **架构设计** | 耦合度、单一职责、可测试性 |
| **性能** | 渲染性能、内存泄漏、打包体积 |
| **安全** | 已知漏洞、敏感信息泄露 |

## 输出模板

```markdown
# 技术债评估报告

## 项目信息
- 项目: <项目名称>
- 评估时间: <日期>
- 评估范围: <目录>

## 总体评分
- 代码质量: X/10
- 架构设计: X/10
- 安全性: X/10
- 性能: X/10
- **综合评分: X/10**

## 问题清单

### 🔴 严重问题 (Critical)
1. ...

### 🟡 重要问题 (Major)
1. ...

### 🟢 一般问题 (Minor)
1. ...

## 改进建议

### 优先级 P0 (立即处理)
1. ...

### 优先级 P1 (本周处理)
1. ...

### 优先级 P2 (下次迭代)
1. ...

## 工作量估算
- 总估算: X 人天
- 建议分 X 个迭代完成
```

## 注意事项

1. 评估应该客观、量化，避免主观判断
2. 提供具体的修复建议，而非泛泛而谈
3. 考虑业务优先级，区分"必须改"和"可以改"
4. 生成可操作的任务清单
